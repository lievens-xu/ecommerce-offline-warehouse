# Spark SQL Migration / Spark SQL 迁移说明

## 1. Purpose / 目的

This document describes the completed Spark SQL migration of the e-commerce offline data warehouse project.

本文档说明电商离线数仓项目已完成 Spark SQL 迁移。

The goal was to add a real SQL execution engine to the project and make the warehouse aggregation logic closer to a big data development workflow.

目标是为项目加入真正的 SQL 执行引擎，使数仓汇总逻辑更加接近大数据开发流程。

The Spark SQL migration demonstrates:

Spark SQL 迁移展示了以下能力：

| Capability / 能力 | Description / 说明 |
|---|---|
| Spark SQL execution / Spark SQL 执行 | Real SQL engine for warehouse transformations / 使用真正的 SQL 引擎执行数仓转换 |
| DWD/DWS/ADS migration / 三层迁移 | Complete migration from Pandas to Spark SQL / 从 Pandas 到 Spark SQL 的完整迁移 |
| Window functions / 窗口函数 | Moving averages, cumulative metrics, day-over-day changes / 移动平均、累计指标、日环比 |
| Automated validation / 自动校验 | Pandas vs Spark comparison across all layers / 跨所有分层的 Pandas 与 Spark 对比 |
| Pipeline orchestration / 流程编排 | One-command Spark migration pipeline / 一键 Spark 迁移主流程 |

## 2. Background / 背景

### 2.1 Original Pandas Baseline / 原始 Pandas 基准

The original version of this project uses Python and Pandas to process data.

本项目原始版本使用 Python 和 Pandas 进行数据处理。

In that version, SQL files are mainly used as Hive-style table design documents.

在该版本中，SQL 文件主要用于表示 Hive 风格的表结构设计。

The Pandas pipeline serves as a stable local baseline and remains unchanged.

Pandas 主流程作为稳定的本地基准，保持不变。

### 2.2 Spark SQL Addition / Spark SQL 新增

The Spark SQL version adds actual SQL execution logic through PySpark.

Spark SQL 版本通过 PySpark 加入了真正的 SQL 执行逻辑。

This migration does not replace the Pandas baseline but adds a parallel Spark migration pipeline.

此迁移不替换 Pandas 基准，而是新增一条并行的 Spark 迁移主流程。

The Spark migration uses the same ODS layer CSV files generated by the Pandas pipeline.

Spark 迁移使用 Pandas 主流程生成的相同 ODS 层 CSV 文件。

### 5.4 Core Transformations / 核心转换

| Transformation / 转换 | Description / 说明 |
|---|---|
| Order items aggregation / 商品聚合 | Aggregate items to order level: item count, product count, seller count, product amount, freight amount / 将商品聚合到订单级：商品数、产品数、卖家数、商品金额、运费 |

| Customer join / 客户关联 | Join customer information: unique ID, city, state / 关联客户信息：唯一 ID、城市、州 |
| Logistics calculation / 物流计算 | Calculate delivery days, estimated days, delay days, is_delivered, is_late_delivery / 计算送达天数、预计天数、延迟天数、是否送达、是否延迟 |

### 7.5 Spark SQL Window Functions / Spark SQL 窗口函数

The ADS transformation uses Spark SQL window functions:

ADS 转换使用 Spark SQL 窗口函数：

**7-day moving average:**

**7 日移动平均：**

```sql
AVG(daily_gmv) OVER (
    ORDER BY stat_date
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
) AS gmv_7d_moving_avg
```

**Cumulative sum:**

**累计求和：**

```sql
SUM(daily_gmv) OVER (
    ORDER BY stat_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
) AS cumulative_gmv
```

**Day-over-day change:**

**日环比变化：**

```sql
daily_gmv - LAG(daily_gmv) OVER (ORDER BY stat_date) AS gmv_day_over_day_change
```
