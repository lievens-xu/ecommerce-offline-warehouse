# Dashboard Design / 看板设计

## 1. Purpose / 目的

This document describes the dashboard design of the e-commerce offline data warehouse project.

本文档说明电商离线数仓项目的看板设计。

The dashboard is built from the ADS layer table:

该看板基于 ADS 应用层表构建：

```
data/ads/ads_daily_business_overview.csv
```

The goal of the dashboard is to visualize key business metrics and make the final data warehouse output easier to understand.

看板的目标是可视化核心业务指标，使最终数仓产出更容易理解和展示。

---

## 2. Dashboard Generation Script / 看板生成脚本

The dashboard charts are generated by:

看板图表由以下脚本生成：

```
scripts/build_dashboard_charts.py
```

Run command:

运行命令：

```
python scripts/build_dashboard_charts.py
```

## 7. Relationship with ADS Layer / 与 ADS 层的关系

The dashboard is directly based on the ADS layer.

看板直接基于 ADS 应用层构建。

The ADS layer already contains dashboard-ready fields such as daily metrics, moving averages, cumulative values, and growth rates.

ADS 层已经包含每日指标、移动平均、累计值和增长率等面向看板的字段。

Therefore, the dashboard generation script mainly focuses on visualization rather than business logic calculation.

因此，看板生成脚本主要关注可视化展示，而不是重复进行业务逻辑计算。

## 8. Project Value / 项目价值

The dashboard module improves the project in three ways:

看板模块从三个方面提升了项目价值：

   它为 GitHub、简历和面试展示提供了可用截图。

## 10. Summary / 总结

The dashboard module is the final presentation layer of this offline data warehouse project.

看板模块是本离线数仓项目的最终展示层。

It transforms ADS metrics into visual business insights, including GMV trends, order trends, AOV trends, logistics performance, review performance, and growth monitoring.

它将 ADS 指标转换为可视化业务洞察，包括 GMV 趋势、订单趋势、客单价趋势、物流表现、评价表现和增长监控。

This module makes the project more complete and easier to present in interviews.

该模块使项目更加完整，也更适合在面试中展示。
